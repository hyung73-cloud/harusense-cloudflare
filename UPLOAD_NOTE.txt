이번 원상복귀 원인 진단
================================================================================
관리자 위치 저장 직후에는 화면의 핀이 새 좌표로 이동합니다.
그런데 이후 지도 데이터가 다시 로드되면서 다음 원본들이 예전 좌표를 다시 가져올 수 있습니다.

- 인라인 지도 스냅샷
- stable/base/override 브라우저 캐시
- /api/clinics 원본 D1 좌표
- 예전 id(newsense)와 현재 id(clinic_12339695) 불일치

그래서 화면 캐시만 지워서는 부족하고, 같은 기관을 나타내는 모든 별칭을 하나로 묶어야 합니다.

이번 보강 내용
================================================================================
1. 12339695 / clinic_12339695 / newsense 를 같은 좌표 그룹으로 취급
2. 인라인 스냅샷 첫 화면에도 저장 좌표를 즉시 적용
3. stable/base/override 캐시가 돌아와도 위치 override를 다시 우선 적용
4. updatePosition API 저장 시 provider_positions 저장
5. 가능하면 D1 원본 clinics/clinic 테이블의 lat/lng 또는 latitude/longitude도 함께 갱신
